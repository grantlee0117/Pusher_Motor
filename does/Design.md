# Pusher Motor 推料电机系统 — 设计文档

## 1. 项目概述

本项目是一个基于 **STM32F103C8T6** 微控制器的推料电机控制系统，通过 PWM 控制直流电机转速实现推料功能。系统同时支持外部硬件触发信号和串口命令行接口（CLI）两种方式来启动电机，所有运行参数均可通过串口实时配置并自动保存到 Flash，确保掉电不丢失。

---

## 2. 硬件平台

| 资源 | 说明 |
|------|------|
| **MCU** | STM32F103C8T6 (Cortex-M3, 72MHz, 64KB Flash, 20KB RAM) |
| **PWM 输出** | TIM4_CH4 (PB9)，控制电机转速 |
| **方向控制 A** | GPIO 输出 (PB8)，**指令控制**一路电机旋转方向 |
| **方向控制 B** | GPIO 输出 (PB7)，**使能控制**一路电机旋转方向 |
| **外部触发** | GPIO 外部中断 (PB6)，上升沿触发 |
| **调试串口** | USART1 (PA9-TX / PA10-RX)，115200 波特率 |

## 4. 核心工作流程

### 4.1 主循环流程

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();          // GPIO + EXTI 中断初始化
    MX_TIM4_Init();          // PWM 定时器初始化
    MX_USART1_UART_Init();   // 调试串口初始化
    
    pusher_motor_init();     // 从 Flash 加载参数，启动 PWM
    cli_init(&huart1);       // 初始化 CLI，打印欢迎信息
    
    while (1)
    {
        cli_process();         // 处理串口输入/输出（非阻塞轮询）
        pusher_motor_loop();   // 执行电机状态机
    }
}
```

主循环以**非阻塞轮询**方式运行，`cli_process()` 和 `pusher_motor_loop()` 每次执行都迅速返回，不占用 CPU 过长时间，确保系统实时响应。

### 4.2 电机控制状态机

电机运行由四级状态机构成：

```
┌─────────┐   start_flag=1   ┌─────────┐
│  IDLE   │ ───────────────> │  WAIT   │
│ (空闲)  │                  │ (等待)  │
└─────────┘                  └─────────┘
     ▲                            │
     │                            │ 等待时间到
     │                            ▼
     │                       ┌─────────┐
     │                       │ RUNNING │
     │                       │ (运行)  │
     │                       └─────────┘
     │                            │
     │                            │ 运行时间到
     │                            ▼
     │                       ┌─────────┐
     └────────────────────── │  STOP   │
                             │ (停止)  │
                             └─────────┘
```

| 状态 | 说明 | PWM 输出 | 持续时间 |
|------|------|----------|----------|
| **IDLE** | 空闲等待，监测启动信号 | 停止（占空比最大 = 高电平） | — |
| **WAIT** | 启动信号到达后的延迟等待 | 停止（占空比最大 = 高电平） | `MOTOR_WAIT_TIME_MS` |
| **RUNNING** | 电机运转，输出 PWM | 运行（按 `MOTOR_PWM_DUTY` 输出） | `MOTOR_DIRECTION_TIME_MS` |
| **STOP** | 运行结束，清理状态回到空闲 | 停止（占空比最大 = 高电平） | 瞬时 |

> **注意**：本项目使用**低电平有效**的 PWM 控制逻辑。占空比数值越小，低电平时间越长，电机转速越快；占空比为最大值（500）时输出恒高电平，电机停止。

### 4.3 启动触发流程

系统支持两种启动方式：

#### 方式一：外部硬件触发
```
外部信号(PB6 上升沿) → EXTI9_5 中断 → HAL_GPIO_EXTI_Callback()
                                            │
                                            ▼
                                  pusher_motor_start()
                                            │
                                            ▼
                         若系统处于 IDLE，则切换为 RUNNING
                                            │
                                            ▼
                                  pusher_motor_start_flag = 1
                                            │
                                            ▼
                              pusher_motor_loop() 状态机检测到标志
                                            │
                                            ▼
                                    进入 WAIT → RUNNING → STOP
```

#### 方式二：串口命令触发
```
用户输入 "start" → USART1 接收 → cli_process() 解析
                                            │
                                            ▼
                                  cli_execute_command(CLI_CMD_START)
                                            │
                                            ▼
                                  pusher_motor_start()
                                            │
                                            ▼
                         若系统处于 IDLE，则切换为 RUNNING
                                            │
                                            ▼
                              pusher_motor_loop() 状态机检测到标志
```

### 4.4 CLI 数据处理流程

```
USART1 硬件接收寄存器
        │
        ▼
cli_receive_data() ──> 环形接收缓冲区 (512B)
        │
        ▼
cli_process() 读取缓冲区
        │
        ▼
process_input_line() 处理行编辑（回显、退格）
        │
        ▼
收到换行符，命令完整
        │
        ▼
parse_command() 解析命令字符串
        │
        ▼
cli_execute_command() 执行对应操作
        │
        ▼
cli_send_string() 发送响应（阻塞方式）
        │
        ▼
输出提示符 "> "
```

CLI 采用**双缓冲机制**：
- **接收环形缓冲区** (`rx_ring_buf`, 512B)：缓存 USART 接收到的原始数据，解决数据突发问题。
- **发送缓冲区** (`tx_buf`, 256B)：缓存回显字符，由 `cli_process_tx()` 在主循环中轮询发送，实现非阻塞回显。

## 5. 参数存储机制

### 5.1 Flash 存储布局

使用 STM32F103C8T6 最后一页 Flash（地址 `0x0800FC00`，每页 1KB）存储参数：

| 偏移 | 字段 | 说明 |
|------|------|------|
| +0 | `direction_time_ms` | 运行时间（ms） |
| +4 | `pwm_duty` | PWM 占空比（0-500） |
| +8 | `wait_time_ms` | 等待时间（ms） |
| +12 | `max_speed_rpm` | 最高转速（RPM） |
| +16 | `motor_mp_a_dir` | A 方向引脚电平 |
| +20 | `motor_mp_b_dir` | B 方向引脚电平 |
| +24 | `checksum` | 校验和（前面 6 个字段之和） |

### 5.2 参数加载流程

```
pusher_motor_init() 调用
        │
        ▼
FlashStorage_Read() 读取 Flash 数据
        │
        ├─── 数据有效（校验和正确、值在范围内）
        │           └──> 使用 Flash 中的参数
        │
        └─── 数据无效（首次烧录、Flash 损坏）
                    └──> 使用默认值，写入 Flash
```

### 5.3 参数修改与保存

当用户通过 CLI 的 `set` 命令修改参数时：
1. 验证参数合法性
2. 更新全局变量
3. 调用 `pusher_motor_save_params()` → `FlashStorage_Write()`
4. FlashStorage 先**擦除整页**，再按字（32-bit）写入新数据
5. 返回操作结果给 CLI，CLI 输出成功/失败提示

## 6. 速度计算模型

系统支持以 **cm/分钟** 为单位的直观速度设置，内部自动转换为 PWM 占空比。

### 6.1 物理模型

- **电机轮直径**：6 cm
- **周长**：`π × 6 ≈ 18.85 cm`
- **线速度与转速关系**：`speed_cm_min = RPM × 周长`

### 6.2 占空比与速度换算

由于 PWM 为**低电平有效**，占空比与转速呈反比关系：

**速度 → 占空比**（`pusher_motor_calculate_duty_from_speed`）：
```
RPM = speed_cm_min / 周长
duty = (MAX_RPM - RPM) / MAX_RPM × MAX_DUTY
```

**占空比 → 速度**（`pusher_motor_calculate_speed_from_duty`）：
```
RPM = (1 - duty / MAX_DUTY) × MAX_RPM
speed_cm_min = RPM × 周长
```

### 6.3 边界条件

| 条件 | 占空比 | 速度 |
|------|--------|------|
| 停止 | 500（最大） | 0 |
| 半速 | 250 | MAX_RPM / 2 |
| 全速 | 0（最小） | MAX_RPM |

## 7. 中断与优先级

| 中断源 | 类型 | 优先级 | 处理函数 | 说明 |
|--------|------|--------|----------|------|
| EXTI9_5 | 外部中断 | 0 (最高) | `HAL_GPIO_EXTI_Callback()` | PB6 上升沿触发，调用 `pusher_motor_start()` |
| SysTick | 系统滴答 | 默认 | `HAL_IncTick()` | 提供 HAL_GetTick() 时基（1ms） |

> 注意：EXTI 中断服务程序仅设置一个标志位，所有实际业务逻辑都在主循环中完成，符合**中断快进快出**原则。

## 8. 关键时序

### 8.1 单次推料时序

```
时间轴(ms)
   0          wait_time          wait_time + direction_time
   │              │                        │
   ▼              ▼                        ▼
   ├──────────────┼────────────────────────┤
   │    等待      │        运行            │
   │  (PWM停止)   │    (PWM输出)           │
   │              │                        │
   │  方向引脚    │     方向引脚           │
   │  保持不变    │     保持不变           │
   │              │                        │
   ▼              ▼                        ▼
  IDLE ──────> WAIT ─────────────────> RUNNING ──> STOP ──> IDLE
```

### 8.2 PWM 时基

- **TIM4 时钟**：APB1 = 36MHz（经倍频后 TIM4 实际时钟为 72MHz）
- **预分频**：72-1 → 计数频率 1MHz
- **周期**：500-1 → PWM 频率 = 1MHz / 500 = **2 kHz**
- **占空比分辨率**：1/500 = 0.2%

## 11. 设计约束与注意事项

1. **实时性**：主循环需保持轻量，`cli_process()` 和 `pusher_motor_loop()` 均不阻塞。
2. **Flash 寿命**：每次 `set` 命令都会擦写 Flash（约 1 万次寿命），生产环境应避免频繁修改参数。
3. **PWM 极性**：本项目使用低电平有效逻辑，占空比越小速度越快，与常规高电平有效逻辑相反。
4. **EXTI 去抖**：PB6 外部触发无硬件去抖，若信号有抖动可能多次触发，需确保上游信号干净。
5. **参数原子性**：Flash 写入过程若掉电，可能导致参数损坏，下次上电将自动恢复默认值。
