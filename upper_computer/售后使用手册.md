# Pusher Motor 售后调试上位机使用手册

本文档面向售后人员。正常使用时不需要理解串口命令和 PWM 原理，只需要按界面按钮操作。

## 1. 现场主要使用步骤

每次到现场调试，优先按下面流程走：

1. 给控制板上电，并把串口线接到电脑。
2. 打开上位机，选择对应 COM 口；没有看到串口就点“刷新”。
3. 点击“连接”，连接成功后右上角会显示“已连接 COMx”。
4. 点击“读取参数”，把设备里当前保存的参数读到界面上。
5. 点击“单次运行测试”，先观察设备当前动作是否正常。
6. 根据现场现象，只调整四个常用参数：等待时间、运行时间、加速度、速度。
7. 每调一次参数，点击“单次运行测试”确认效果。
8. 效果合适后，点击对应“保存”按钮。
9. 最后再点一次“读取参数”，确认界面显示的是刚保存的值。

如果现场只是临时试速度，速度可以先点“应用”；如果要断电后继续保留，必须点“保存”。

## 2. 售后主要调哪几个参数

售后日常一般只需要调下面四个参数：

| 参数 | 主要作用 | 什么时候调 | 工程默认值 |
| --- | --- | --- | --- |
| 等待时间 | 收到启动信号后，延迟多久再开始动 | 动作开始太早或太晚 | 250 ms |
| 运行时间 | 电机真正运行多久 | 输送距离不够或过多 | 250 ms |
| 加速度 | 启动时速度爬升快慢 | 启动太冲或启动太慢 | 100 |
| 速度 | 传送带运行快慢 | 输送太快或太慢 | 约 48226 cm/min |

不建议售后日常修改“方向”“PWM”“速度校准 RPM”。这些属于高级调试，通常只有研发或明确指导时才改。

## 3. 四个常用参数怎么理解

### 3.1 等待时间 ms

含义：设备收到启动信号后，先等多久再让电机开始运行。

默认值：`250 ms`

范围：`0-10000 ms`

调节方法：

- 动作开始太早：增大等待时间。
- 动作开始太晚：减小等待时间。
- 希望收到信号后马上动作：可以设为 `0`。

修改后点击“保存”。保存后会写入设备，断电后仍然保留。

### 3.2 运行时间 ms

含义：电机开始运行后，持续运行多久。

默认值：`250 ms`

范围：`1-60000 ms`

调节方法：

- 输送距离不够：增大运行时间。
- 输送距离过多：减小运行时间。

修改后点击“保存”。保存后会写入设备，断电后仍然保留。

### 3.3 加速度 0-100

含义：电机启动时，速度从慢到快的过程有多快。界面数值越大，启动越快。

默认值：`100`

范围：`0-100`

调节方法：

- 启动太冲、机械冲击明显：减小加速度，例如从 `100` 调到 `70` 或 `60`。
- 启动太慢、拖得太久：增大加速度。

本质说明：固件内部实际保存的是“加速时间”，但售后界面已经换算成更直观的“加速度”。售后只需要记住：数值越大，启动越快。

修改后点击“保存”。保存后会写入设备，断电后仍然保留。

### 3.4 速度 cm/min

含义：传送带运行速度，单位是厘米/分钟。数值越大，速度越快。

工程默认值：约 `48226 cm/min`

这个默认值由底层默认 PWM `150` 和速度校准值 `3655 RPM` 换算得到。售后不需要直接理解 PWM，正常调“速度”即可。

界面上有两个按钮：

| 按钮 | 作用 | 是否断电保留 |
| --- | --- | --- |
| 应用 | 临时试一下当前速度 | 不保证断电保留 |
| 保存 | 保存为设备默认速度 | 断电后保留 |

推荐用法：

1. 调整速度数值。
2. 点击“应用”。
3. 点击“单次运行测试”观察效果。
4. 速度合适后点击“保存”。

## 4. 现场现象和参数对应关系

| 现场现象 | 优先调整 |
| --- | --- |
| 收到信号后动作太早 | 增大等待时间 |
| 收到信号后动作太晚 | 减小等待时间 |
| 推送距离不够 | 增大运行时间 |
| 推送距离过多 | 减小运行时间 |
| 输送太慢 | 增大速度 |
| 输送太快 | 减小速度 |
| 启动瞬间冲击明显 | 减小加速度 |
| 启动太慢 | 增大加速度 |

建议一次只改一个参数。改完先做“单次运行测试”，确认效果后再继续调下一个参数。

## 5. 保存和断电保留规则

简单记法：

- 点“保存”的参数，会写入设备，断电后保留。
- 点“应用”的速度或 PWM，主要用于临时测试。
- “读取参数”只读取，不修改设备。
- “单次运行测试”只触发一次动作，不修改参数。

保存成功后，界面会弹出“保存成功”，约 1 秒后自动消失。

## 6. 快速恢复工程默认值

如果现场参数被调乱，可以按下面值手动恢复：

| 参数 | 恢复值 |
| --- | --- |
| 等待时间 | 250 |
| 运行时间 | 250 |
| 加速度 | 100 |
| 速度 | 约 48226 |
| 电机A | 1 |
| 电机B | 0 |

恢复后分别点击对应“保存”按钮。

如果高级调试里的 PWM 被改乱，可把 PWM 改回 `150` 并点击“保存”。

## 7. 常用动作说明

- “读取参数”：从设备读取当前保存的参数，并刷新界面显示。
- “单次运行测试”：让设备按当前参数执行一次动作，用来验证输送效果。
- “读取启动信号”：读取外部启动输入脚当前电平。
- “启动信号”：显示上一次读取到的外部启动输入状态。

“启动信号”显示含义：

| 显示 | 含义 |
| --- | --- |
| HIGH | 外部启动输入当前为高电平，设备可被外部信号触发 |
| LOW | 外部启动输入当前为低电平，没有检测到外部启动输入 |

## 8. 顶部状态说明

顶部显示当前连接状态和系统模式。

常见模式说明：

| 显示 | 含义 | 售后处理方式 |
| --- | --- | --- |
| IDLE | 空闲，可以调参和测试 | 正常 |
| RUNNING | 电机正在执行动作 | 等动作结束后再操作 |
| SERVICE | 售后临时调试状态 | 正常，程序会自动进入或退出 |
| ERROR | 异常状态 | 断电重启后重新连接；仍异常则反馈研发 |

售后不需要手动切换模式。

## 9. 使用前准备

### 9.1 需要准备

- 已烧录匹配固件的控制板。
- Windows 电脑。
- USB 转串口工具或设备自带串口连接线。
- 本目录下的上位机程序文件。

### 9.2 串口参数

程序默认串口参数如下，一般不需要修改：

| 项目 | 默认值 |
| --- | --- |
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |

### 9.3 启动程序

如果是源码方式运行：

```bat
py -m pip install -r requirements.txt
py app.py
```

如果已经打包成 exe，直接双击 `PusherMotorHost.exe`。

## 10. 方向参数

方向参数用于设置电机方向控制引脚电平。

| 参数 | 默认值 |
| --- | --- |
| 电机A | 1 |
| 电机B | 0 |

一般情况下不要修改方向参数。只有出现电机方向反了、接线或机构需要调整时，才根据现场情况修改。

修改后分别点击“保存A”或“保存B”。保存后会写入设备，断电后仍然保留。

## 11. 高级调试

“高级调试”默认收起。普通售后调试不建议使用，除非研发明确要求。

### 11.1 PWM 0-500

PWM 是底层速度控制值。这个项目里 PWM 和速度是同一件事的两种表达方式：

- 售后界面里的“速度”更直观。
- 高级调试里的“PWM”更偏底层。

注意：本固件里 PWM 数值越大，速度越慢；PWM 数值越小，速度越快。

默认 PWM：`150`

按钮说明：

| 按钮 | 作用 |
| --- | --- |
| 应用 | 临时设置 PWM，不保存 |
| 保存 | 保存 PWM，断电后保留 |

普通售后优先调“速度”，不要直接调 PWM。

### 11.2 速度校准 RPM

默认值：`3655 RPM`

作用：用于“速度 cm/min”和底层 PWM 之间的换算。

一般不要修改。只有研发确认电机、轮径、传动结构或测速基准变更后，才需要改这个值。

## 12. 常见问题处理

### 12.1 找不到串口

处理步骤：

1. 检查串口线是否插好。
2. 检查控制板是否上电。
3. 点击“刷新”。
4. 打开 Windows 设备管理器，确认是否出现 COM 口。
5. 如果没有 COM 口，检查 USB 转串口驱动。

### 12.2 连接失败

处理步骤：

1. 确认没有其他串口工具占用该 COM 口。
2. 确认波特率是 `115200`。
3. 断开后重新连接。
4. 控制板断电重启后再试。

### 12.3 点击按钮后提示超时

可能原因：

- 串口线松动。
- 控制板没有上电。
- COM 口选错。
- 固件版本和上位机不匹配。

处理步骤：

1. 断开上位机连接。
2. 检查接线和上电。
3. 重新选择串口并连接。
4. 仍无法恢复时，把通信日志截图发给研发。

### 12.4 单次运行测试没有动作

处理步骤：

1. 确认设备已连接。
2. 点击“读取参数”，看是否能正常读取。
3. 查看顶部模式是否为 `RUNNING`，如果正在运行，等待结束。
4. 检查速度是否被设置得过低。
5. 检查运行时间是否太短。
6. 检查电机供电、驱动和接线。

### 12.5 外部启动没有反应

处理步骤：

1. 点击“读取启动信号”。
2. 触发外部启动源后再读取一次。
3. 如果一直是 `LOW`，检查外部启动信号线。
4. 如果能变成 `HIGH`，但设备仍不运行，反馈研发排查固件状态和电机驱动。

## 13. 打包说明

在 Windows 环境中，进入 `upper_computer` 目录，双击：

```bat
build_windows.bat
```

默认会打包成单文件 exe，程序在：

```text
dist\PusherMotorHost.exe
```

把这个 `PusherMotorHost.exe` 发给同事即可，对方不需要安装 Python 或依赖库。

注意：不要把 `dist\PusherMotorHost\` 文件夹里的单独 exe 拿出来发给别人。文件夹模式必须发送整个文件夹，否则对方容易出现“加载 Python DLL 失败”。

打包前建议先运行一次源码版本，确认串口连接和按钮功能正常。

## 14. 给研发排查用的信息

售后遇到问题时，建议提供：

- 设备编号或现场信息。
- 使用的上位机版本或文件日期。
- 控制板固件版本或烧录日期。
- Windows 设备管理器里的 COM 口截图。
- 上位机右侧“通信日志”截图。
- 当前参数截图。
- 问题发生前点过哪些按钮。

正常售后使用不需要阅读底层串口命令。如需查命令细节，可参考项目根目录的 `串口调试命令.md`。
