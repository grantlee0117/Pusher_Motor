# Pusher Motor 售后调试上位机使用手册

## 出发前准备

到现场之前，确认带齐以下物品：

- 已烧录匹配固件的控制板
- Windows 电脑
- USB 转串口工具或设备自带串口连接线
- 本目录下的上位机程序文件（源码或打包好的 `PusherMotorHost.exe`）

程序默认串口参数如下，一般不需要修改：

| 项目 | 默认值 |
|------|--------|
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |

---

## 第一章 快速上手

每次到现场，不需要从头到尾读完整本手册。跟着下面四步走，五分钟就能完成一次完整的调试。

![上位机界面概览](figures/ui-overview.png)
*图 1-1：上位机界面概览。左侧为参数调节区，右侧为通信日志区。*

### 1.1 连接设备

1. 给控制板上电，用串口线把控制板和电脑连好。
2. 打开上位机程序，在左上方的"串口"下拉框里选对应的 COM 口。如果列表是空的，点一下旁边的"刷新"。
3. 波特率保持默认的 `115200`，一般不需要改。
4. 点"连接"。连接成功后，右上角状态会变成"已连接 COMx"，同时显示当前设备模式。

![连接区域](figures/ui-connection.png)
*图 1-2：连接区域。选择串口、设置波特率、点击连接。*

如果连不上，见附录 B.2"连接失败"。

### 1.2 读取参数

连接成功后，第一步永远是点"读取参数"。

这个按钮会把设备里当前保存的参数全部读到界面上。只有先读取，你看到的数值才是设备里真实存的值，而不是界面上的默认值。

读取完成后，界面会弹出"读取完成"的提示。

### 1.3 单次运行测试

参数读上来之后，点"单次运行测试"。

设备会按当前参数执行一次完整的推送动作。观察传送带的实际运行效果：

- 动了吗？
- 开始得早还是晚？
- 跑的距离够吗？
- 启动瞬间冲不冲？
- 速度快不快？

把观察到的问题记下来，下一步就是针对性调参。

![常用动作区域](figures/ui-quick-actions.png)
*图 1-3：常用动作区域。读取参数、单次运行测试、读取启动信号。*

### 1.4 调参与保存

根据刚才观察到的问题，改对应的参数。改完后，再点一次"单次运行测试"确认效果。

效果合适了，点对应参数的"保存"按钮。保存成功后界面会弹出"保存成功"，约 1 秒后自动消失。

**一个原则：一次只改一个参数。** 改完测完，再改下一个。如果一次改好几个，出了问题不知道是谁引起的。

**另一个原则：调参阶段先用"应用"试，确认了再"保存"。** 速度和 PWM 这两个参数有"应用"按钮。点"应用"后参数临时生效，断电后不保留。试好了再点"保存"，才真正写进设备。

> **注意"应用"的两点限制：**
> 1. "应用"只能在设备空闲（IDLE）时使用。如果设备正在运行（RUNNING），点"应用"会报错拒绝。
> 2. 点"应用"后，设备会自动进入 SERVICE 模式。这是正常的临时调试状态，不需要手动切换。

保存完，建议最后再点一次"读取参数"，确认界面上显示的就是你刚保存的值。

## 第二章 四个参数详解

售后日常只需要调四个参数：等待时间、运行时间、加速度、速度。这一章逐个讲清楚每个参数"是什么""背后在调节什么""怎么调"。

### 2.1 等待时间

**是什么：** 设备收到外部启动信号后，延迟多久才让电机开始转动。

**背后在调节什么：**

现场通常有传感器或其他设备给控制板发启动信号。但这个信号到达的时机，和传送带真正需要开始动作的时机，不一定完全同步。等待时间就是用来对齐这两个时机的缓冲。

你可以把它理解为"听到指令后倒数几秒再起跑"。倒数的秒数太长，起跑就晚了；太短，可能还没准备好就开跑。

- **默认值：** `250 ms`
- **可调范围：** `0 ~ 10000 ms`
- **怎么调：**
  - 动作开始太早（还没准备好就动了）→ **增大**等待时间
  - 动作开始太晚（信号来了半天才动）→ **减小**等待时间
  - 希望收到信号后马上动作 → 设为 `0`

改完后点"保存"。

![运行参数区域](figures/ui-runtime-params.png)
*图 2-1：运行参数区域。运行时间和等待时间的滑块与保存按钮。*

### 2.2 运行时间

**是什么：** 电机从开始转动到停止，一共持续多久。

**背后在调节什么：**

运行时间直接决定传送带的输送距离。电机转速不变的情况下，转得越久，送得越远。这个参数本质上是在控制"一次推送要把物料送多远"。

- **默认值：** `250 ms`
- **可调范围：** `1 ~ 60000 ms`
- **怎么调：**
  - 输送距离不够（物料没送到位置）→ **增大**运行时间
  - 输送距离过多（物料送过头了）→ **减小**运行时间

改完后点"保存"。

### 2.3 加速度

**是什么：** 电机启动时，从静止加速到目标速度的快慢程度。界面显示 `0 ~ 100`，数值越大，启动越快。

**背后在调节什么：**

电机从静止瞬间跳到全速，会产生很大的机械冲击，长期下去可能损坏传送机构或让物料移位。加速度参数控制的就是这个"起步过程有多柔和"。

这里需要理解一点：界面上显示的"加速度"和固件底层的"加速时间"大致对应，但不是严格的一一换算。

- 界面显示 `100` → 固件加速时间约为 `0 ms`，意味着几乎没有加速过程，直接全速启动
- 界面显示 `0` → 固件加速时间约为 `10000 ms`，意味着用 10 秒钟慢慢爬到目标速度

所以售后只需要记住：**数值越大，启动越冲；数值越小，启动越柔和。**

- **默认值：** `100`（无加速，直接全速启动）
- **可调范围：** `0 ~ 100`
- **怎么调：**
  - 启动瞬间冲击明显（机械振动大、噪音大）→ **减小**加速度，例如从 `100` 调到 `70` 或 `60`
  - 启动太慢、拖得太久才达到正常速度 → **增大**加速度

改完后点"保存"。

![速度参数区域](figures/ui-speed-params.png)
*图 2-2：速度参数区域。速度（应用/保存双按钮）、加速度（保存按钮），均带滑块。*

### 2.4 速度

**是什么：** 传送带正常运行时的速度，单位是厘米/分钟（cm/min）。数值越大，跑得越快。

**背后在调节什么：**

速度决定的是单位时间内传送带走了多远。和运行时间配合起来，两者共同决定一次推送的总距离。

简单理解：
- 速度快 + 运行时间不变 → 送得更远
- 速度慢 + 运行时间不变 → 送得更近

界面上显示的速度值，是上位机帮你换算好的直观数值。底层其实用的是 PWM（脉宽调制）来控制电机转速，但售后不需要直接操作 PWM——调"速度"更直观。

默认速度 `48226 cm/min` 是从底层默认 PWM `150` 和速度校准值 `3655 RPM` 换算得到的。售后日常只需要关心界面上的"速度"数值，不需要管 PWM。

- **默认值：** `48226 cm/min`
- **可调范围：** `0 ~ 70000 cm/min`

> **注意：** 速度设为 `0` 时，电机不会转动。如果点了"单次运行测试"但设备没动作，先检查一下速度是不是被误设成了 `0`。

**操作方式（重点）：**

速度参数有两个按钮，用法和其他参数不一样：

| 按钮 | 作用 | 断电后是否保留 |
|------|------|--------------|
| 应用 | 临时试一下当前速度 | 不保留 |
| 保存 | 保存为设备默认速度 | 保留 |

**推荐用法：**

1. 在速度输入框里填入想试的数值。
2. 点"应用"。此时速度临时生效，但还没有写进设备。
3. 点"单次运行测试"，观察效果。
4. 速度合适后，点"保存"。保存成功后才真正写入设备，断电后仍然保留。

**怎么调：**
- 输送太慢 → **增大**速度
- 输送太快 → **减小**速度

## 第三章 现场速查

这一章不追求从头到尾读，而是放在手边，遇到问题直接翻到对应条目。

### 3.1 现象→参数对照表

| 现场现象 | 优先调整的参数 | 调节方向 |
|---------|--------------|---------|
| 收到信号后动作太早 | 等待时间 | 增大 |
| 收到信号后动作太晚 | 等待时间 | 减小 |
| 推送距离不够 | 运行时间 | 增大 |
| 推送距离过多 | 运行时间 | 减小 |
| 输送太慢 | 速度 | 增大 |
| 输送太快 | 速度 | 减小 |
| 启动瞬间冲击明显（振动/噪音大） | 加速度 | 减小 |
| 启动太慢、拖太久 | 加速度 | 增大 |

**原则：一次只改一个参数，改完先做"单次运行测试"确认效果，再改下一个。**

### 3.2 设备状态说明

界面右上角显示当前设备模式，不同模式的含义和处理方式：

| 显示 | 含义 | 售后处理方式 |
|------|------|------------|
| IDLE | 空闲，可以调参和测试 | 正常操作 |
| RUNNING | 电机正在执行动作 | 等动作结束后再操作 |
| SERVICE | 售后临时调试状态 | 正常，程序会自动进入或退出 |
| ERROR | 异常状态 | 断电重启后重新连接；仍异常则反馈研发 |

售后不需要手动切换模式。点"应用"速度或 PWM 时，程序会自动进入 SERVICE 模式；保存参数后会自动回到 IDLE。

![状态显示](figures/ui-status.png)
*图 3-1：界面顶部状态栏。显示连接状态和当前设备模式。*

### 3.3 应用 vs 保存

| 操作 | 作用 | 是否写入设备 | 断电后是否保留 |
|------|------|-----------|--------------|
| 保存 | 把参数写进设备 | 是 | 是 |
| 应用（仅限速度、PWM） | 临时生效试一下 | 否 | 否 |
| 读取参数 | 从设备读当前值到界面 | 否 | — |
| 单次运行测试 | 触发一次动作 | 否 | — |

简单记法：
- 点"保存"的参数，会写入设备，断电后保留。
- 点"应用"的速度或 PWM，主要用于临时测试。只能在 IDLE 状态下使用，会自动进入 SERVICE 模式。
- "读取参数"只读取，不修改设备。
- "单次运行测试"只触发一次动作，不修改参数。

![保存成功提示](figures/ui-toast.png)
*图 3-2：保存成功后的提示弹窗，约 1 秒后自动消失。*

### 3.4 恢复工程默认值

如果现场参数被调乱，可以按下面值手动恢复：

| 参数 | 恢复值 |
|------|--------|
| 等待时间 | 250 ms |
| 运行时间 | 250 ms |
| 加速度 | 100 |
| 速度 | 48226 cm/min |
| 电机 A | 1 |
| 电机 B | 0 |

恢复方法：在对应输入框填入上表数值，分别点击对应参数的"保存"按钮。

如果高级调试里的 PWM 被改乱，可把 PWM 改回 `150` 并点击"保存"。

## 第四章 扩展功能

### 4.1 方向参数

方向参数控制电机转动方向的引脚电平。

| 参数 | 默认值 |
|------|--------|
| 电机 A | 1 |
| 电机 B | 0 |

**一般情况下不要修改方向参数。** 只有在现场发现电机实际转动方向和预期相反时，才考虑调整。通常是接线或机械安装的问题，优先排查硬件；确认需要改方向时，在下拉框选 `0` 或 `1`，然后点"保存A"或"保存B"。

保存后写入设备，断电保留。

![方向区域](figures/ui-direction.png)
*图 4-1：方向参数区域。电机 A 和电机 B 的方向设置与保存按钮。*

### 4.2 启动信号读取

点击"读取启动信号"，可以查看外部启动输入脚当前的电平状态。

| 显示 | 含义 |
|------|------|
| HIGH | 外部启动输入为高电平，设备可被外部信号触发 |
| LOW | 外部启动输入为低电平，没有检测到外部启动输入 |

如果外部启动没有反应，可以用这个按钮排查：先点一次，记下显示的状态；然后触发外部启动源，再点一次，对比两次结果。如果两次都是 LOW，说明信号没有送到控制板，检查外部信号线。如果第二次变成了 HIGH，但设备仍不运行，反馈研发排查固件状态和电机驱动。

## 第五章 高级调试

高级调试区域默认是收起的，点"高级调试"按钮才会展开。**普通售后调试不建议使用，除非研发明确要求。**

![高级调试区域](figures/ui-advanced.png)
*图 5-1：高级调试区域（展开状态）。包含 PWM 和速度校准 RPM。*

### 5.1 PWM

PWM（脉宽调制）是底层速度控制值。它和界面上的"速度"是同一回事的两种表达方式：

- "速度"更直观，售后日常用这个。
- "PWM"更底层，数值范围和物理含义不那么直观。

**注意：本固件里 PWM 数值越大，速度越慢；PWM 数值越小，速度越快。** 这和直觉相反，容易搞混，所以售后优先调"速度"，不要直接调 PWM。

- **默认值：** `150`
- **可调范围：** `0 ~ 500`

| 按钮 | 作用 |
|------|------|
| 应用 | 临时设置 PWM，不保存 |
| 保存 | 保存 PWM，断电后保留 |

### 5.2 速度校准 RPM

- **默认值：** `3655 RPM`
- **作用：** 用于"速度 cm/min"和底层 PWM 之间的换算

一般不要修改。只有研发确认电机型号、轮径、传动结构或测速基准发生变更后，才需要改这个值。

## 附录 B 常见问题处理

### B.1 找不到串口

1. 检查串口线是否插好。
2. 检查控制板是否上电。
3. 点击"刷新"。
4. 打开 Windows 设备管理器，确认是否出现 COM 口。
5. 如果没有 COM 口，检查 USB 转串口驱动。

### B.2 连接失败

1. 确认没有其他串口工具占用该 COM 口。
2. 确认波特率是 `115200`。
3. 断开后重新连接。
4. 控制板断电重启后再试。

### B.3 点击按钮后提示超时

可能原因：串口线松动、控制板没有上电、COM 口选错、固件版本和上位机不匹配。

处理步骤：
1. 断开上位机连接。
2. 检查接线和上电。
3. 重新选择串口并连接。
4. 仍无法恢复时，把通信日志截图发给研发。

### B.4 单次运行测试没有动作

1. 确认设备已连接。
2. 点击"读取参数"，看是否能正常读取。
3. 查看顶部模式是否为 `RUNNING`，如果正在运行，等待结束。
4. 检查速度是否被设置得过低（包括是否误设为 `0`）。
5. 检查运行时间是否太短。
6. 检查电机供电、驱动和接线。

### B.5 外部启动没有反应

1. 点击"读取启动信号"。
2. 触发外部启动源后再读取一次。
3. 如果一直是 `LOW`，检查外部启动信号线。
4. 如果能变成 `HIGH`，但设备仍不运行，反馈研发排查固件状态和电机驱动。

## 附录 C 打包说明

在 Windows 环境中，进入 `upper_computer` 目录，双击：

```bat
build_windows.bat
```

默认会打包成单文件 exe，程序在：

```text
dist\PusherMotorHost.exe
```

把这个 `PusherMotorHost.exe` 发给同事即可，对方不需要安装 Python 或依赖库。

**注意：** 不要把 `dist\PusherMotorHost\` 文件夹里的单独 exe 拿出来发给别人。文件夹模式必须发送整个文件夹，否则对方容易出现"加载 Python DLL 失败"。

打包前建议先运行一次源码版本，确认串口连接和按钮功能正常。
