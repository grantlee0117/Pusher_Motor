# 做事记录 · 总摘要

## 项目演进概述

本项目从最初的一个简单推料电机控制需求，逐步演进为一个功能完整的嵌入式控制系统。

### 阶段一：最小可用原型（MVP）

初始目标非常简单：基于 STM32F103C8T6，检测 PB7 输入信号，通过 PB8 控制方向、PB9 输出 PWM，驱动电机运行固定时长后停止。参数通过宏定义管理，修改参数需要重新编译烧录。

### 阶段二：功能扩展

随着调试和集成需求的出现，系统逐步增加了以下能力：

1. **串口命令行接口（CLI）**：为便于现场调试，添加了 USART1 串口通信，支持通过命令实时修改参数，无需重新编译。
2. **Flash 参数持久化**：参数修改后自动保存到 Flash，断电不丢失，避免了每次上电重新配置的麻烦。
3. **参数管理器（params_manager）**：将分散的参数管理逻辑抽取为独立模块，统一负责参数的读写、有效性检查和持久化，提高了代码的可维护性。
4. **状态机完善**：将简单的工作流程扩展为四状态机（IDLE → WAIT → RUNNING → STOP），增加了启动前等待延迟功能。
5. **速度换算功能**：基于电机直径 6cm，实现了速度（cm/min）与 PWM 占空比之间的相互换算，方便操作人员按工艺需求直接设置线速度。
6. **双路方向控制**：由单路方向控制扩展为 A/B 两路方向控制（PB8/PB7），适应实际硬件接线。
7. **外部中断触发**：将启动信号检测方式改为 PB6 外部中断（上升沿触发），提高了触发实时性，同时保留了串口命令启动方式。
8. **临时调试命令**：增加了 `set new_pwm_duty` 命令，可直接修改 PWM 占空比但不保存到 Flash，方便现场快速试凑最优参数。
