# Pusher Motor 推料电机项目说明

## 项目概述

本项目是一个基于 STM32F103C8T6 微控制器的推料电机控制系统，通过 PWM 控制电机正反转，实现推料和回位功能。项目包含硬件控制逻辑和串口命令行接口（CLI），支持通过串口实时调整运行参数。

## 系统架构

### 硬件平台
- **MCU**: STM32F103C8T6 (Cortex-M3, 72MHz)
- **PWM 输出**: TIM4_CH4 (PB9)
- **方向控制**: GPIO 输出 (PB8)
- **启动信号**: GPIO 输入中断 (PB7)
- **调试串口**: USART1 (PA9/PA10)

## 核心模块说明

### 1. 推料电机控制模块 (pusher_motor.c)

#### 功能
- 初始化 PWM 输出和 GPIO 方向控制
- 实现推料和回位动作控制
- 状态机管理电机运行状态

#### 关键参数
```c
uint32_t MOTOR_DIRECTION_TIME_MS = 1500;  // 每个方向运行时间（毫秒）
uint32_t MOTOR_PWM_DUTY = 500;            // PWM 占空比（0-1000）
```

#### 状态机设计
```
┌─────────┐    start     ┌───────────┐
│  IDLE   │ ───────────> │  FORWARD  │
│ (空闲)  │              │  (正转)   │
└─────────┘              └─────┬─────┘
     ▲                         │
     │                         │ 时间到
     │                         ▼
     │                   ┌───────────┐
     │                   │  REVERSE  │
     │                   │  (反转)   │
     │                   └─────┬─────┘
     │                         │
     │                         │ 时间到
     │                         ▼
     └────────────────── ┌───────────┐
                         │   STOP    │
                         │  (停止)   │
                         └───────────┘
```

#### 工作流程
1. **触发启动**：外部中断（PB7 高电平）或 CLI 命令触发 `pusher_motor_start()`
2. **正转阶段**：设置方向引脚为高电平，启动 PWM 输出，运行设定时间
3. **反转阶段**：切换方向引脚为低电平，继续 PWM 输出，运行相同时间
4. **停止阶段**：停止 PWM 输出，回到空闲状态

### 2. CLI 命令行接口模块 (cli.c)

#### 功能
- 串口命令接收和解析
- 实时修改电机运行参数
- 命令回显和响应输出

#### 接收机制
```
串口数据 → 环形缓冲区 → 命令解析 → 执行处理 → 响应输出
```

#### 环形缓冲区设计
```c
#define RX_BUF_SIZE 512
static uint8_t rx_ring_buf[RX_BUF_SIZE];  // 接收环形缓冲区
static volatile uint16_t rx_head = 0;     // 写指针
static volatile uint16_t rx_tail = 0;     // 读指针
```

#### 支持的命令
| 命令 | 功能 |
|------|------|
| `start` | 启动推料电机 |
| `set direction_time <ms>` | 设置方向运行时间 |
| `set pwm_duty <value>` | 设置 PWM 占空比 |
| `get direction_time` | 获取当前方向运行时间 |
| `get pwm_duty` | 获取当前 PWM 占空比 |
| `help` | 显示帮助信息 |

### 3. 中断处理模块 (stm32f1xx_it.c)

#### 外部中断 EXTI9_5
- **触发源**：PB7 引脚（推料电机启动信号）
- **触发方式**：上升沿触发
- **处理逻辑**：调用 `pusher_motor_start()` 启动推料

## 数据流

### 推料触发流程
```
外部信号(PB7) ──> EXTI 中断 ──> pusher_motor_start() 
                                          │
                                          ▼
                              ┌─────────────────────┐
                              │   pusher_motor_loop │
                              │   (主循环调用)      │
                              └─────────────────────┘
```

### CLI 命令流程
```
串口输入 ──> 环形缓冲区 ──> 命令解析 ──> 参数修改/电机控制
                │                           │
                ▼                           ▼
           字符回显                    响应输出
```

## 主循环逻辑

```c
int main(void)
{
    // 初始化
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    MX_TIM4_Init();          // PWM 定时器
    MX_USART1_UART_Init();   // 调试串口
    
    pusher_motor_init();     // 初始化推料电机
    cli_init(&huart1);       // 初始化 CLI
    
    while (1)
    {
        // 处理推料电机状态机
        pusher_motor_loop();
        
        // 处理 CLI 命令
        cli_process();
    }
}
```

## 关键时序

### PWM 输出
- **频率**：由 TIM4 配置决定（通常 1kHz-20kHz）
- **占空比**：0-1000 对应 0%-100%
- **输出引脚**：PB9 (TIM4_CH4)

### 推料周期
```
时间(ms)
  0    ┌───────────┐
       │   正转    │

 3000  └───────────┘           ├──────┐
       │           │           │ 停止 │
       └───────────┴───────────┘      │
                                      ▼
```

## 扩展接口

### 添加新命令
在 `cli.c` 中添加：
1. 在 `CliCommand_t` 枚举中添加新命令类型
2. 在 `parse_command()` 中添加命令解析逻辑
3. 在 `cli_execute_command()` 中添加命令执行代码

### 修改默认参数
在 `pusher_motor.c` 中修改：
```c
uint32_t MOTOR_DIRECTION_TIME_MS = 1500;  // 修改默认值
uint32_t MOTOR_PWM_DUTY = 500;            // 修改默认值
```

## 调试方法

1. **串口调试**：连接 USART1，使用 115200 波特率
2. **逻辑分析仪**：监测 PB7（触发）、PB8（方向）、PB9（PWM）
3. **断点调试**：在关键状态转换处设置断点

## 注意事项

1. **中断优先级**：确保 EXTI 中断优先级合理，不影响主循环
2. **缓冲区溢出**：环形缓冲区满时会丢弃新数据
3. **参数有效性**：CLI 会检查参数范围，无效参数会返回错误
4. **实时性**：主循环需要快速执行，避免阻塞
