# Pusher Motor CLI 命令文档

## 简介

本文档描述了通过串口 1 控制推料电机的 CLI（命令行界面）命令。这些命令允许您启动推料电机、修改和读取推料电机的运行参数。

## 串口连接信息

- **串口**：USART1
- **波特率**：115200
- **数据位**：8
- **停止位**：1
- **校验位**：无
- **流控制**：无
- **GPIO 引脚**：
  - TX: PA9
  - RX: PA10

## 可用命令

### 1. 启动推料电机

```
start
```

启动推料电机，执行推料操作（正转 1.5 秒，然后反转 1.5 秒）。

**示例**：

```
> start
Pusher motor started.
>
```

### 2. 设置方向运行时间

```
set direction_time <ms>
```

设置推料电机每个方向的运行时间（以毫秒为单位）。

**参数**：

- `<ms>`: 运行时间，单位为毫秒（必须为正整数）

**示例**：

```
> set direction_time 2000
Direction time set to 2000 ms
>
```

### 3. 设置 PWM 占空比

```
set pwm_duty <value>
```

设置推料电机的 PWM 占空比。

**参数**：

- `<value>`: PWM 占空比，范围为 0-500

**示例**：

```
> set pwm_duty 500
PWM duty set to 500
>
```

### 4. 获取当前方向运行时间

```
get direction_time
```

显示当前设置的方向运行时间。

**示例**：

```
> get direction_time
Current direction time: 1500 ms
>
```

### 5. 获取当前 PWM 占空比

```
get pwm_duty
```

显示当前设置的 PWM 占空比。

**示例**：

```
> get pwm_duty
Current PWM duty: 500
>
```

### 6. 帮助命令

```
help
```

显示所有可用的命令及其描述。

**示例**：

```
> help
Available commands:
  start - Start pusher motor
  set direction_time <ms> - Set direction time in milliseconds
  set pwm_duty <value> - Set PWM duty cycle
  get direction_time - Get current direction time
  get pwm_duty - Get current PWM duty cycle
  help - Show this help
>
```

## 默认参数

- **方向运行时间**：1500 毫秒（1.5 秒）
- **PWM 占空比**：500

## 注意事项

1. **命令格式**：命令区分大小写，请使用小写字母输入命令。
2. **参数范围**：
   - `direction_time` 必须为正整数（单位：毫秒）
   - `pwm_duty` 范围为 0-1000
3. **推料流程**：当执行 `start` 命令时，推料电机会先正转设定的时间，然后反转相同的时间，最后停止。
4. **实时修改**：在推料电机运行过程中修改参数不会立即生效，需要等待当前运行周期结束后才会应用新的参数。
5. **回显功能**：输入的字符会实时回显到串口，方便查看输入内容。
6. **退格支持**：支持退格键（Backspace）删除输入的字符。

## 故障排除

### 命令无响应

- **原因**：可能是串口连接不正确，或者波特率设置错误。
- **解决方案**：检查串口连接，确保波特率设置为 115200。

### 命令被识别为未知命令

- **原因**：可能是命令拼写错误，或者命令格式不正确。
- **解决方案**：检查命令拼写，确保使用正确的命令格式。可以使用 `help` 命令查看可用命令。

### 推料电机不运行

- **原因**：可能是 PWM 占空比设置为 0，或者方向运行时间设置为 0。
- **解决方案**：检查并修改 PWM 占空比和方向运行时间的设置。

## 技术细节

### 串口接收机制

CLI 使用非阻塞轮询方式接收串口数据：

1. 在主循环中快速检查串口接收寄存器
2. 将接收到的数据存入环形缓冲区
3. 当检测到回车或换行符时，解析并执行命令
4. 使用环形缓冲区处理突发数据，避免数据丢失

### 命令解析流程

1. 接收字符并回显
2. 存储字符到命令缓冲区
3. 检测到回车或换行符时，结束命令接收
4. 解析命令和参数
5. 执行对应的命令处理函数
6. 发送响应信息
7. 发送提示符，等待下一次命令
