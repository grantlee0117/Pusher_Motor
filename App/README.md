# App 目录

`App/` 是手写业务代码区。CubeMX 重新生成代码时，不应覆盖这里。

- `System/`: 应用入口，负责组合初始化和主循环调度
- `Control/`: 电机控制、状态机、运动参数计算
- `Communication/`: 串口 CLI 和后续协议处理
- `Storage/`: Flash、EEPROM、外部存储等持久化封装
- `Config/`: 应用参数结构、校验、加载和保存策略

新增功能优先放在这里；只有外设初始化、中断模板、HAL 配置才放回 `Core/`。

## 分层约定

- `System/` 负责把 HAL 句柄和业务模块连接起来，不承载复杂业务规则。
- `Control/` 负责电机状态机、运动参数和控制行为。
- `Communication/` 负责串口命令解析和对外交互格式。
- `Storage/` 负责持久化细节，业务模块不直接操作 Flash。
- `Config/` 负责参数结构、默认值、范围和校验策略。

修改 `App/` 代码时，需要同步检查根目录 README 和 `docs/串口调试命令.md` 是否仍然准确。
